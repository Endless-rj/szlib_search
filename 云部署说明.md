# 深圳图书馆搜索 - 云部署说明

## 项目说明

`szlib_cloud.py` 是一个**完全自包含的单文件 Flask 应用**，无需任何外部静态文件（index.html、manifest.json、sw.js 等），可直接部署到各大云平台。

- **唯一依赖**：`flask`（Python 标准库 `urllib` 无需安装）
- **适用场景**：云平台部署（本地/Termux 场景请使用 `szlib_lite.py`）

## 部署到 Railway（免费额度）

### 前提条件

- 注册 [Railway](https://railway.app/) 账号（可用 GitHub 登录）

### 部署步骤

**方式一：从 GitHub 仓库部署**

1. 登录 Railway 控制台，点击 **New Project**
2. 选择 **Deploy from GitHub repo**
3. 选择包含 `szlib_cloud.py` 的仓库

**方式二：使用空白模板部署**

1. 点击 **New Project** → **Blank template**
2. 在项目页面点击 **Settings** → **Environment**
3. 也可直接拖拽上传文件

### 配置服务

1. 在项目页面点击新建的服务，进入 **Settings** 标签
2. 配置构建与启动命令：
   - **Build Command**：`pip install flask`
   - **Start Command**：`python szlib_cloud.py`
3. 确保仓库根目录包含 `requirements.txt`，内容如下：
   ```
   flask
   ```
   Railway 会自动检测并安装依赖。

### 生成域名

1. 在服务设置中找到 **Networking** 或 **Domains**
2. 点击 **Generate Domain**
3. Railway 会分配一个 `xxx.up.railway.app` 域名
4. 使用生成的域名即可访问应用

## 解决境外服务器连接问题（Cloudflare Worker 代理）

> Railway 的境外服务器无法直接连接 `szlib.org.cn`，这是网络层面的问题。可通过 **Cloudflare Workers 做代理**来巧妙解决。

### 部署 Cloudflare Worker 代理

1. 打开 [workers.cloudflare.com](https://workers.cloudflare.com)
2. 用 GitHub 或 Google 登录（免费，无需信用卡）
3. 点击 **Create Worker**
4. 给 Worker 起个名字，比如 `szlib-proxy`
5. 把编辑器里的代码全部删掉，粘贴 `worker.js` 的内容，点击 **Save and Deploy**
6. 得到 Worker 地址，格式如：`https://szlib-proxy.你的名字.workers.dev`

### 验证代理是否正常

浏览器打开以下地址（**注意：需要翻墙才能访问，内地无法直接访问 Cloudflare 链接**）：

```
https://szlib-proxy.你的名字.workers.dev/api/opacservice/getQueryResult?library=all&v_tablearray=bibliosm,&sortfield=ptitle&sorttype=desc&pageNum=1&v_page=1&v_secondquery=&client_id=t1&v_index=title&v_value=%E7%BA%A2%E6%A5%BC%E6%A2%A6
```

如果看到 JSON 数据，说明代理工作正常！

### 在 Railway 中配置代理

1. 打开你的 Railway 项目
2. 点击 **Variables** 标签
3. 添加环境变量：
   - **Key**：`PROXY_URL`
   - **Value**：`https://szlib-proxy.你的名字.workers.dev`（即上一步得到的 CF Worker 地址）
4. Railway 会自动重新部署
